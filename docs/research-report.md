## 1. 거래소 API 조사 결과

### 업비트 (Upbit) - 메인 거래소로 선정

| 항목 | 내용 |
|------|------|
| REST API | `https://api.upbit.com/v1/` — 시세, 주문, 계좌 전부 지원 |
| WebSocket | `wss://api.upbit.com/websocket/v1` — 실시간 틱커/호가/체결 |
| Rate Limit | 시세 10req/s, 주문 8req/s (충분) |
| Python 라이브러리 | **pyupbit** — 활발히 유지보수, 한국 커뮤니티 대규모 |
| 인증 방식 | JWT (API Key + Secret Key) |
| 제약 사항 | 실명 인증 필수, IP 화이트리스트 설정 필요 |

### 코인원 (Coinone) - 보조/비교용

| 항목 | 내용 |
|------|------|
| REST API | 있음, 기본 기능 지원 |
| Python 라이브러리 | 공식 없음. 직접 래퍼 작성 필요 |
| 문서 품질 | 업비트 대비 부족 |
| 거래량 | 업비트의 20-30% 수준 |

**결론: 업비트를 메인으로 사용.** pyupbit 라이브러리, 문서 품질, 거래량, 커뮤니티 모두 압도적. 코인원은 김치프리미엄 교차 검증용으로만 활용.

## 2. 매매 전략 초기 추천안

### 2-1. 매매 스타일: 스윙 트레이딩 (4시간봉 기준)

| 스타일 | 적합성 | 이유 |
|--------|--------|------|
| 스캘핑 | **부적합** | 수수료 0.05~0.25%, 왕복 0.1~0.5%. 스캘핑 목표 수익(0.1~0.3%)을 수수료가 전부 먹음 |
| 데이트레이딩 | 보통 | 목표 0.5~2%, 수수료 비중 5~50%. 마진 얇음 |
| **스윙** | **최적** | 목표 3~15%, 수수료 비중 1~5%. 크립토 높은 변동성과 잘 맞음 |

**구체 파라미터:**
- 메인 타임프레임: **4시간봉** (신호 생성), 1시간봉 (진입 타이밍 미세 조정)
- 보유 기간: 1~5일
- 트레이드당 목표: 3~8%
- 주간 최대 거래: 5~8회

### 2-2. 기술적 지표 조합

```
매수 조건 (모두 충족 시):
  ① EMA(9)가 EMA(21)을 상향 돌파 (4시간봉)
  ② RSI(14)가 35~65 사이 (과매수 구간 진입 방지)
  ③ 거래량 > 20기간 평균 거래량의 1.5배 (거래량 확인)
  ④ 가격 > EMA(50) (대추세 필터: 상승 추세에서만 매수)

매도 조건 (하나라도 충족 시):
  ① EMA(9)가 EMA(21)을 하향 돌파
  ② 손절선 도달 (2x ATR)
  ③ 익절선 도달 (3x ATR 또는 트레일링)
```

**지표 선택 근거:**
- **EMA 9/21**: SMA보다 크립토의 빠른 움직임에 민감. 4시간봉 기준 약 1.5일/3.5일 룩백
- **RSI(14)**: 과매수/과매도 필터. 크립토에서 높은 타임프레임 RSI가 가장 안정적
- **거래량**: 거래량 동반 돌파의 후속 추세 유지율 ~65% vs 미동반 ~45%
- **ATR(14)**: 변동성 적응형 손절/익절. 시장 상황에 자동 대응
- **제외 항목**: MACD(EMA 기반이라 중복), 볼린저밴드(V2에서 평균회귀 전략으로 추가), 일목균형표(자동화 복잡)

### 2-3. 손절/익절

| 항목 | 설정 |
|------|------|
| 손절 | **2x ATR(14)** — BTC 기준 보통 3~5%, 알트코인 5~10% |
| 절대 손절 | **-8%** 하드캡 (급락 보호) |
| 익절 초기 목표 | **3x ATR** (최소 Risk:Reward 1.5:1) |
| 트레일링 | 2x ATR 수익 도달 시 손절을 본전으로 이동, 이후 1.5x ATR로 추적 |
| 시간 손절 | 48시간 내 ±1% 이내 → 시장가 매도 (데드 포지션 정리) |

**Risk:Reward 1.5:1 근거:**
- 승률 50%일 때: (0.5 × 1.5) - (0.5 × 1.0) = **+0.25** (양의 기대값)
- 승률 45%일 때: (0.45 × 1.5) - (0.55 × 1.0) = **+0.125** (여전히 수익)

### 2-4. 동시 포지션 수

| 원금 | 최대 포지션 수 | 포지션당 금액 | 이유 |
|------|---------------|-------------|------|
| 10만원 | **1개** | 10만원 | 분할 시 수수료 비중 과다 |
| 50만원 | **2개** | ~25만원 | |
| 100만원 | **2~3개** | ~33만원 | |

**자금 배분 원칙:**
- 단일 포지션 최대: 전체 자산의 40%
- 전체 투자 비중 최대: 70% (항상 30% 현금 보유)
- 트레이드당 리스크: 전체 자산의 **1%**

## 3. 거시경제 지표 연동

### Tier 1: 매매 신호에 직접 반영

| 지표 | 상관관계 | 활용 방법 |
|------|---------|----------|
| **BTC 도미넌스** | 매우 높음 | 상승 시 BTC만 거래, 하락 시 알트코인 거래 |
| **공포탐욕지수** | 높음 | <20 매집, >80 비중 축소, 20~80 정상 운영 |
| **김치프리미엄** | 높음 | >5% 약세 신호(소매 과열), <0% 강세 신호(항복) |
| **달러인덱스(DXY)** | 높음(역상관) | 강한 상승 시 크립토 비중 축소. 2022~2024 상관계수 -0.7~-0.85 |

### Tier 2: 포지션 사이징에 반영 (이벤트 기반)

| 지표 | 활용 방법 |
|------|----------|
| **FOMC 금리 결정** | 인상: 포지션 50% 축소(48시간), 인하: 비중 확대, 동결: 정상 |
| **CPI/PPI** | 높은 CPI: 24~48시간 방어적, 낮은 CPI: 위험자산 강세 |

### Tier 3: 대시보드 표시만 (자동 매매 미반영)

| 지표 | 이유 |
|------|------|
| 고용지표(비농업 고용) | 크립토 영향 간접적, 상관관계 비일관적 |
| 스테이블코인 유입/유출 | 유용하지만 자동화 어려움 |
| 거래소 보유량 | 온체인 지표, 변화 느림, 주간 편향 참고용 |

## 4. 뉴스/이벤트 기반 트레이딩

**결론: 직접 매매 신호가 아닌 "리스크 필터"로만 사용**

**이유:**
- NLP 감성 분석의 방향 예측 정확도 ~55~60% (수수료 차감 후 랜덤에 가까움)
- 기관 트레이더(Jump, Wintermute 등)이 뉴스에 밀리초 단위로 반응 — 소매 봇은 속도 경쟁 불가
- 크립토 뉴스의 90%는 가격 영향 없음. 영향 있는 10%는 60초 내 반영 완료

**구현:**
```
리스크 필터 (포지션 축소 또는 매매 일시 중단):
  - FOMC 결정, CPI 발표, 한국 규제 발표: 전후 2시간 신규 진입 금지
  - 공포탐욕지수 < 10 또는 > 95: 신규 진입 중단
  - BTC 24시간 내 -10% 이상 하락: 6시간 신규 진입 중단 (연쇄 청산 리스크)
```

**V2에서 추가:** 이벤트 캘린더 (FOMC, CPI, 한국 규제 회의, 주요 토큰 언락 일정) → 자동으로 리스크 파라미터 조정

## 5. AI/ML vs 룰 기반

**결론: 룰 기반으로 시작 → 단계적 ML 도입**

| 기준 | 룰 기반 | ML |
|------|---------|-----|
| 필요 데이터 | 최소 | 2년+ 클린 데이터 |
| 디버깅 | 투명 | 블랙박스 |
| 과적합 위험 | 낮음 | 매우 높음 |
| 개발 기간 | 일~주 | 주~월 |
| 소자본 적합성 | **높음** | 낮음 (학습 기간 동안 자본 손실 위험) |

**3단계 접근:**

1. **Phase 1 (지금)**: 순수 룰 기반 — EMA + RSI + 거래량. 모든 매매 데이터 꼼꼼히 기록
2. **Phase 2 (200+ 거래 데이터 축적 후)**: ML로 시장 레짐 분류 (추세/횡보/고변동) → 레짐에 따라 전략 자동 전환
3. **Phase 3 (500+ 거래 후, 선택)**: 강화학습으로 포지션 사이징 최적화

**LLM 활용:** Claude API로 뉴스/소셜 감성 분석 가능. 뉴스 리스크 필터에 적합. 단, 직접 매매 신호보다는 위험도 점수 산출용으로 활용.

## 6. 코인 선별 기준

### 제외 필터 (하나라도 해당 시 제외)

| 필터 | 기준 | 이유 |
|------|------|------|
| 일일 KRW 거래량 | < 10억원 | 유동성 부족, 스프레드 큼, 조작 쉬움 |
| 시가총액 | < $50M | 조작 위험, 러그풀 영역 |
| 상장 기간 | < 6개월 | 분석할 가격 이력 부족 |
| 호가 스프레드 | > 0.3% | 슬리피지가 수익 잠식 |
| 가격 | < 10 KRW | 틱 사이즈 문제, 비정상적 % 변동 |

### 예측 가능성 점수 (남은 코인 순위 매김)

| 지표 | 가중치 | 설명 |
|------|--------|------|
| 거래량 일관성 | 30% | 30일 일일 거래량의 변동계수(CV) < 1.0 |
| 기술적 반응성 | 30% | 90일간 EMA 크로스 전략 백테스트, Sharpe > 0.5 |
| BTC 상관관계 | 20% | 30일 롤링 상관 0.5~0.8이 최적 |
| 변동성 | 20% | 연환산 60~150%가 최적 |

### 시작 코인 풀

- **항상 포함:** BTC/KRW, ETH/KRW (가장 유동적, 데이터 풍부, 예측 가능)
- **후보 알트코인:** XRP, SOL, DOGE, ADA, AVAX, MATIC, LINK (업비트 거래량 상위)
- **주간 갱신:** 매주 일요일 필터 재적용 → 상위 5~8개 코인 = 해당 주 허용 목록

## 7. 리스크 관리

### 최대 손실 한도

| 레벨 | 기준 | 조치 |
|------|------|------|
| 소프트 한도 | 전체 자산 **-10%** | 포지션 사이즈 50% 축소, 전략 점검 |
| 하드 한도 | 전체 자산 **-15%** | **전면 중단 72시간** |
| 일일 손실 한도 | 당일 **-3%** | 당일 잔여 시간 매매 중단 |

### 일일 거래 제한

- 하루 최대 **신규 진입 3건**
- 코인당 하루 최대 **1건**
- 과매매(overtrading)는 소자본 계좌 파산의 1번 원인

### 서킷 브레이커 (시장 급변 대응)

```
Level 1 - 주의 (자동):
  조건: BTC 4시간 내 -5% 이상 하락
  조치: 전 포지션 50% 정리, 4시간 신규 진입 중단

Level 2 - 방어 (자동):
  조건: BTC 24시간 내 -10% 이상 하락 OR 일일 손실 -2%
  조치: 전 포지션 청산, 12시간 매매 중단

Level 3 - 비상 (자동):
  조건: BTC 24시간 내 -15% 이상 하락 OR 피크 대비 -10% 드로다운
  조치: 전 포지션 청산, 100% 현금, 72시간 중단, 알림 발송

Level 4 - 킬 스위치 (수동):
  조건: API 에러, 비정상 동작, 수동 개입
  조치: 전 포지션 시장가 즉시 매도, 봇 비활성화
```

### 포지션 사이징 공식

```
position_size = (자본금 × 트레이드당_리스크%) / 손절_비율%

예시: 100만원, 리스크 1%, 손절 5%
position_size = (1,000,000 × 0.01) / 0.05 = 200,000원

최대 포지션 = min(계산된_사이즈, 자본금 × 0.40)
```

## 10. 백테스팅 방침

사용자 요구: **"실제 돈이 나간다는 생각으로, 실제처럼 시뮬레이션. 테스트 데이터 기반으로 발전."**

### 백테스팅 원칙

1. **리얼리스틱 시뮬레이션**
   - 수수료 포함 (업비트 0.05% 매수/매도)
   - 슬리피지 시뮬레이션 (호가창 깊이 기반)
   - 주문 체결 지연 반영 (시장가: 즉시, 지정가: 미체결 가능)
   - 최소 주문 금액 5,000 KRW 반영

2. **데이터 기반 진화**
   - 모든 백테스트 결과 DB에 저장
   - 파라미터 변경은 한 번에 하나씩 (과학적 실험 설계)
   - 핵심 지표: 승률, 평균 R:R, 최대 낙폭, 샤프 비율, 수익 팩터
   - 최소 12~24개월 데이터로 검증

3. **페이퍼 트레이딩 → 실전**
   - 백테스트 통과 후 2~4주간 실시간 페이퍼 트레이딩
   - 페이퍼 결과가 백테스트와 일치하면 소액(10만원)으로 실전 시작
   - 점진적 증액
