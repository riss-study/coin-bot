**목적**: Week 2 데이터 확장 범위(BTC 외 알트)의 **사전 지정 선정 기준**과 **측정 방법**을 박제하는 단일 문서. W2-01.2/.3에서 실측 후 확정 리스트 및 common-window 시작일을 같은 파일에 추가 기록.

- **임계값 완화 금지**: 후보 수가 부족해도 기준 변경은 새 사전 등록 사이클(=새 W2-01) 필요.

## 1. 선정 기준 (3개, AND 조건)

후보 페어는 아래 3개 기준을 **모두 동시 충족**해야 한다.

### 기준 1 — 시가총액 (CoinGecko 글로벌)

| 항목 | 값 |
|------|-----|
| **임계값** | CoinGecko KRW 환산 시총 **상위 30위 이내** |
| **스냅샷 시각** | **2026-04-17 00:00 UTC** (W1-06 No-Go 결정일 자정) |
| **API 엔드포인트** | `GET https://api.coingecko.com/api/v3/coins/markets` |
| **파라미터** | `vs_currency=krw`, `order=market_cap_desc`, `per_page=30`, `page=1` |
| **응답 개수 검증** | `len(response) == 30` assert 필수. 불일치 시 W2-01.2 중단 후 사용자 보고 (API 스펙 변경 가능성). 재조회는 사용자 승인 후 단일 재시도에 한함 |
| **저장 위치** | `research/data/coingecko_top30_snapshot_20260417.json` (git tracked, 원본 응답 + `snapshot_utc` + `fetched_at` 필드 포함) |
| **무결성** | SHA256을 `research/data/data_hashes.txt`에 기록. W2 실행 중 재조회 금지 |
| **근거** | 글로벌 유동성 + 시총 필터. 30위 컷오프는 업계 통념(메이저 알트 범위)과 Week 2 실험 스코프(6-18셀) 양립 |

### 기준 2 — 업비트 KRW 페어 상장일

| 항목 | 값 |
|------|-----|
| **임계값** | 상장일 **≤ 2023-04-17** (W1-06 결정일 기준 정확히 3년 전) |
| **측정 방법** | (1) **업비트 공식 공지 우선** (공지 URL + 내용 캡처를 `research/data/upbit_listing_dates_20260417.json`에 병기 저장). (2) 공지 소실 시 `pyupbit.get_ohlcv_from(ticker="KRW-XXX", interval="day", fromDatetime="2017-01-01 00:00:00", to=None)` 호출 후 `df.index.min()`을 상장일로 채택. `fromDatetime`을 페어 상장 추정일보다 최소 3년 이른 2017-01-01로 고정하여 "페이지네이션 종료 지점이 상장일로 오인"되는 것을 방지. Sanity check: 최초 30 캔들 사이 7일 이상 갭 없음 |
| **저장 위치** | W2-01.2 조회 후 페어별 상장일을 `research/data/upbit_listing_dates_20260417.json`(git tracked, SHA256 기록) + 이 문서 섹션 5 실측 표에 동시 기록 |
| **근거** | 5년 advertised 범위(2021-01-01 ~ 2026-04-12) 중 **60% 이상(3년 이상) 커버** 보장. 짧은 역사 페어는 common-window로 curve fitting 여지 확대 |

### 기준 3 — 업비트 KRW 30 UTC-day 단순 평균 거래대금

| 항목 | 값 |
|------|-----|
| **임계값** | 측정 창 내 30 UTC-day 단순 평균 **≥ 100억 원** |
| **측정 창** | **2026-03-13 ~ 2026-04-11 (UTC, inclusive 양끝, 정확히 30 UTC days)**. W1 freeze 시점 2026-04-12 직전 |
| **산식 (기본)** | `daily_value_i = close_i × volume_i` (일봉 OHLCV 종가 × 거래량, KRW 단위). 이는 **종가 기준 근사 거래대금**. 측정 창 내 30개 값의 **산술 평균**: `Σ(daily_value_i) / 30`, i = 2026-03-13 ~ 2026-04-11 UTC 일별 인덱스 |
| **산식 (우선 사용)** | 업비트 일봉 API 응답에 실제 거래대금 필드(`value` 또는 `candle_acc_trade_price`)가 존재하면 **해당 필드를 직접 사용**(일중 가격 변동으로 인한 근사 오차 제거). W2-01.1 이행 시점에 pyupbit 응답 스키마를 확인하여 정확한 필드명 채택 + 본 문서 갱신. 경계 페어(99~102억)에서 근사 vs 실측 차이가 결론을 바꾸지 않도록 **실측 필드 우선**. 단 필드 부재가 확인되면 기본 산식(`close × volume`)을 채택하고 그 사실을 evidence에 기록 |
| **"rolling" 단어 금지** | pandas `.rolling(30).mean()`은 각 날짜마다의 이동 평균 벡터를 생성. 본 기준은 **단일 창의 단일 평균값** 하나만 요구. 구현 시 `df.loc["2026-03-13":"2026-04-11", "daily_value"].mean()` 형태 |
| **데이터 출처** | 업비트 일봉(`pyupbit.get_ohlcv`, interval=`day`) — 본 프로젝트가 freeze할 동일 소스 |
| **금지 사항** | CoinGecko 24h 스팟 거래대금 **사용 금지** (노이즈 크고 후보별 유리한 창을 사후 선택할 여지) |
| **100억 임계값 근거** | W2 실험 slippage 허용 수준 잠정 추정. 업비트 KRW 시총 상위 20위 알트의 2025 거래대금 분포 중앙값 수준. |
| **100억 sanity check 정책** | W2-01.2 실측 후 상위 20위 중앙값 대비 괴리 판단: ① **±30% 이내**: 임계값 100억 유지, 후보 리스트 그대로 진행. ② **±30% 초과**: 사용자에게 보고하되 **W2-01 현 사이클에서는 100억 그대로 유지하여 W2-01.2~.7 완주**. 임계값 변경 의사 결정은 별도 새 사전 등록 사이클(새 W2-01')로만 가능. 현 사이클 중 임계값 변경은 금지 |

## 2. Tier 분류

### Tier 선정 규칙 (사전 지정)

Tier 1/2는 아래 규칙으로 사전 지정되었다. **이 리스트는 freeze** — 사후 대체 페어 추가 금지.

- **Tier 1 (필수 포함, primary Go 대상)**: BTC (W1 재사용) + ETH (글로벌 시총 2위, 업비트 상장 장기).
- **Tier 2 (exploratory 후보)**: 선정 규칙 = "**CoinGecko 2026-04-17 00:00 UTC 스냅샷 기준 시총 상위 10위 이내** ∩ **업비트 KRW 페어 존재** ∩ **BTC/ETH 제외**". 2026-04-17 시점 사용자 승인 리스트 = {XRP, SOL, ADA, DOGE}. **이 리스트는 기준 승인 시점 freeze**. W2-01.2 실측에서 2026-04-17 스냅샷 원본 JSON에 대해 위 규칙을 재계산하여 리스트 {XRP, SOL, ADA, DOGE}와 불일치가 확인되면 → **현 W2-01 사이클 중단 + Fallback (ii) 재설계**. 실측 결과로 리스트를 사후 교체 금지.

규칙은 W2 실행 중 변경 불가. 새 리스트 사용 시 새 W2-01 사이클 필요.

## 3. Fallback 정책 (Tier 2 <2 통과 시)

외부 감사 BLOCKING B-5 해소. **임계값 완화는 절대 금지**. Tier 2 통과 페어가 2개 미만(=0 또는 1)이면 아래 두 옵션 중 택일, 사용자 명시적 승인 필수.

| 옵션 | 내용 | 영향 |
|------|------|------|
| **(i) Tier 2 제거** | Tier 1 × {A, C, D} = **primary 6셀 그대로 유지**. Tier 2 exploratory는 통과 페어 수 × 3 전략으로 축소 (0개 통과 → exploratory 0셀, 1개 통과 → exploratory 3셀). Go 기준 Sharpe>0.8 AND DSR>0 **변경 없음** | W2-03 sub-plan에서 exploratory 범위만 실측 수에 맞게 조정. primary 구조는 불변 |
| **(ii) Task 재설계** | W2-01 재시작. 새 사전 등록 문서 + backtest-reviewer + 사용자 승인 루프. 기존 W2-01의 실측 결과는 "참고 자료"로 격리, primary Go 평가에 반영 금지 | Week 2 일정 +2일 이상 지연. 재설계 시 임계값 변경은 가능하지만 **새 사이클**로만 가능 |

**금지 사항**:
- 기준 임계값(30위, 2023-04-17, 100억)을 "하나만 통과"로 완화 금지
- Tier 2 후보 리스트 {XRP, SOL, ADA, DOGE} 이외 페어(LINK, SHIB, AVAX 등)를 사후 추가 금지 (Tier 2 통과 2개 이상이어도 동일)
- "거의 충족"한 페어를 포함 금지 (예: 상장 2023-05-01 페어를 "비슷하니까" 승격 금지, 99.5억 거래대금 페어를 "반올림 시 100억" 승격 금지)
- Fallback (i) 선택 시 Go 기준(Sharpe>0.8 AND DSR>0)이나 primary 셀 구성 변경 금지

## 4. Common-window 사전 결정 (W-2 대응)

### 배경

Tier 2 후보 중 일부(SOL, DOGE 등)는 W1 advertised 범위(2021-01-01)보다 업비트 KRW 상장일이 늦을 가능성이 있다. 페어별 실제 범위가 다르면 백테스트 지표(Sharpe, PF 등)의 직접 비교가 불공정. 실제 상장일은 W2-01.2에서 확정.

### 결정

1. W2-01.2 실측 후 **Tier 1-2 최종 페어 중 상장일이 가장 늦은 페어**의 업비트 KRW 상장일을 common-window 시작점으로 채택.
2. W2-03 grid에서 **두 metric을 동시 계산**:
   - **Primary metric**: 페어별 max-span Sharpe (각 페어 자체 상장일부터 **2026-04-12 UTC**까지 = W1 freeze 종료일과 동일)
   - **Secondary metric**: common-window Sharpe (공통 시작점부터 **2026-04-12 UTC**까지)
3. Go 기준 평가는 **primary metric** 기준. Secondary는 페어 간 비교 공정성 확인용.
4. Common-window 시작일은 W2-01.2 실측 완료 후 이 문서 "5. 확정 리스트" 섹션에 기록 + freeze.

**참고 (종료일 정의)**: 백테스트 metric 산출 종료일(`2026-04-12 UTC`)은 W1 advertised freeze 종료일과 동일(sub-plan L144 `RANGE = ("2021-01-01", "2026-04-12")`와 일치). 기준 3 측정 창 종료일(`2026-04-11 UTC`)은 freeze 직전 30일의 마지막 날로 별개 목적. 두 날짜가 1일 차이나는 것은 의도된 설계.

### 예상 시나리오 (예시, 실제 상장일은 W2-01.2에서 확정)

| 시나리오 | Tier 2 최종 구성 | Common-window 시작일 |

**산출 규칙**: Tier 1-2 최종 리스트 중 **상장일이 가장 늦은 페어의 업비트 KRW 상장일**을 common-window 시작일로 채택. 실제 상장일은 업비트 공식 공지 또는 `pyupbit.get_ohlcv_from` 최초 캔들로 확정한다.

### 갭 처리 사전 정책

Common-window 내 거래정지(업비트 공지성 정지 또는 네트워크 이슈)로 인한 결측 캔들 처리 규칙을 사전 박제한다.

1. **Forward-fill 금지**. 누락 일자를 직전 값으로 채워 "거래가 계속된 것처럼" 만드는 처리는 백테스트 결과를 왜곡하므로 금지.
2. **결측일은 `return=0` + 포지션 상태 유지**. 해당 일자 트레이드 실행 불가로 간주, 이전 포지션 그대로. 결측일 일일 수익률 기여 0.
3. **3일 초과 연속 갭 페어는 플래그**. W2-01.5 무결성 검증 단계에서 `research/outputs/pair_gap_report.md` 별도 표시. W2-03 metric 산출 시 해당 페어 신뢰구간 축소 표기(Sharpe ± bootstrap SE 등).
4. **상장폐지 확정 페어**: Tier 2 기준 충족했더라도 common-window 내 영구 상장폐지 이력 있으면 즉시 탈락 (Fallback 정책 발동 여부 판정).

세부 구현은 W2-03 sub-plan이 본 규정을 준수하여 작성.

발효 후 W2 실행(W2-01 ~ W2-03) 종료까지 아래 변경 금지:

- 기준 3개(시총 상위 30위, 상장 ≤ 2023-04-17, 30 UTC-day 평균 ≥ 100억) 임계값 변경 금지

- CoinGecko 스냅샷 재조회 금지 (JSON 원본 + SHA256으로 freeze)

- Fallback 정책 옵션 변경 금지

### 6.2 확정 리스트 freeze (섹션 5 실측 결과 표 + 최종 확정 값)
