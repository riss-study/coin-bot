## 1. 트레이딩 기본 용어

### 현물 (Spot)
- **뜻**: 코인을 진짜로 사고파는 거래. 산 코인이 진짜 내 지갑에 들어옴.
- **반대**: 선물(Futures) — 코인을 안 사고 "오를지 내릴지" 베팅. 레버리지 가능, 위험.
- **우리 프로젝트**: 현물만 함. 선물은 절대 안 함.

### 포지션 (Position)
- **뜻**: 지금 보유 중인 코인. "BTC 포지션 0.01개" = "비트코인 0.01개 들고 있음".
- **자바 비유**: 객체가 메모리에 살아있는 상태. 포지션을 "오픈"하면 객체 생성, "클로즈"하면 객체 소멸.

### 진입 (Entry) / 청산 (Exit)
- **진입**: 코인을 사기 시작 (포지션 오픈)
- **청산**: 코인을 다 팔아서 포지션 정리

### 매수 / 매도 (Buy / Sell)
- 그냥 사기 / 팔기

### 손절 (Stop Loss)
- **뜻**: "이 가격까지 떨어지면 손해 보더라도 무조건 판다"는 자동 매도 규칙.
- **예**: 비트코인 1억원에 샀는데 손절을 9200만원으로 설정 → 9200만원 되면 자동 매도, -8% 손실 확정.
- **왜**: 더 떨어져서 -50% 같은 큰 손실 막으려고.
- **자바 비유**: try-catch로 예외 처리하듯이, 매매에서 "최악의 시나리오 안전장치".

### 익절 (Take Profit)
- 손절의 반대. "이 가격까지 오르면 무조건 이익 챙기고 판다."

### 트레일링 스톱 (Trailing Stop)
- **뜻**: 가격이 오르면 손절선도 같이 따라 올라가는 동적 손절.
- **예**: 1억원에 사고 손절 9000만원. 가격이 1억1천이 되면 손절선이 9900만원으로 자동 상향. 떨어지면 9900만원에 청산 → 손해 안 보고 이익 보존.
- **자바 비유**: 옵저버 패턴 — 가격 변화에 손절선이 자동 반응.

### 레버리지 (Leverage)
- **뜻**: 빌린 돈으로 더 큰 거래. "10배 레버리지" = 100만원으로 1000만원어치 거래.
- **위험**: 10% 손실만 나도 원금 100% 손실.
- **우리 프로젝트**: 절대 안 씀.

### 슬리피지 (Slippage)
- **뜻**: 주문 낸 가격과 실제 체결 가격의 차이.
- **예**: "1억원에 살게요" 주문 → 실제로는 1억50만원에 체결됨 → 슬리피지 50만원 (0.5%)
- **왜 발생**: 주문하는 사이에 가격이 움직이거나, 호가창에 그 가격에 매물이 부족해서.
- **자바 비유**: HTTP 요청을 보낸 시점과 서버가 받은 시점 사이의 latency. 그 사이에 데이터가 변할 수 있음.

### 호가창 (Orderbook)
- **뜻**: 매수/매도 주문이 가격대별로 쌓여 있는 표.
- **자바 비유**: 우선순위 큐(Priority Queue) 두 개 — 매수 큐(높은 가격이 우선), 매도 큐(낮은 가격이 우선). 두 큐의 머리가 만나면 거래 체결.

### 시장가 / 지정가 (Market / Limit)
- **시장가**: "지금 가격이 얼마든 무조건 사" → 즉시 체결, 슬리피지 큼
- **지정가**: "1억원이 되면 사" → 그 가격이 될 때까지 대기, 안 될 수도 있음

### 수수료 (Fee)
- **뜻**: 거래소에 내는 거래 수수료
- **업비트 KRW 페어**: 0.05% (매수) + 0.05% (매도) = 왕복 0.1%
- **영향**: 자주 거래하면 수수료가 수익을 다 잡아먹음. 그래서 우리는 "스윙" 전략(자주 거래 안 함)을 함.

### 스캘핑 / 데이트레이딩 / 스윙
- **스캘핑**: 1분~수분 단위로 자주 사고팜. 수수료에 잡아먹힘 (우리 프로젝트에선 부적합).
- **데이트레이딩**: 하루 안에 사고팜
- **스윙(Swing)**: 며칠~몇 주 보유. 우리가 채택한 방식.

### 페어 (Pair)
- **뜻**: "BTC/KRW" = "비트코인을 원화로 사고팜"
- "ETH/USDT" = "이더리움을 USDT(달러 스테이블코인)로 사고팜"
- 우리는 KRW 페어만 거래.

### 페이퍼 트레이딩 (Paper Trading)
- **뜻**: 진짜 돈 안 쓰고 가짜 계좌로 시뮬레이션하면서 진짜 시장 데이터 받음.
- **자바 비유**: 스테이징 환경. 코드는 프로덕션과 동일, DB만 가짜.

### 라이브 트레이딩 (Live Trading)
- 페이퍼의 반대. 진짜 돈으로 진짜 거래.

## 2. 기술적 지표 (매매 신호 만드는 숫자들)

기술적 지표 = 가격/거래량 데이터에서 계산한 숫자. 매매 결정에 사용.

### MA (Moving Average, 이동평균선)
- **뜻**: "최근 N일 종가의 평균"
- **MA200**: 최근 200일 평균 종가 → 장기 추세 판단
- **예**: 오늘 비트코인 1억원, MA200 = 9000만원 → "현재가가 200일 평균보다 높음 → 상승 추세"
- **사용**: "현재가 > MA200이면 상승장이니까 매수만 한다 (하락장에는 매수 안 함)"
- **자바 비유**: `list.subList(size-200, size).stream().mapToDouble(Candle::close).average()`

### EMA (Exponential Moving Average, 지수이동평균)
- **뜻**: MA와 비슷하지만 최근 데이터에 가중치를 더 줌.
- **MA vs EMA**: MA는 모든 일자 동등 가중치. EMA는 어제가 1주일 전보다 더 중요.

### RSI (Relative Strength Index, 상대강도지수)
- **뜻**: 0~100 사이 값. 최근 N일 동안 "오른 날 vs 내린 날" 비율로 계산.
- **해석**:
  - **RSI > 70**: 너무 많이 올랐음 (과매수, overbought)
  - **RSI < 30**: 너무 많이 떨어졌음 (과매도, oversold)
- **사용**: "RSI < 25면 과매도라 곧 반등할 거다. 사자."
- **RSI(4)**: 최근 4일 데이터로 계산. 짧을수록 민감, 길수록 부드러움.
- **우리 프로젝트**: Strategy B에서 "RSI(4) < 25 → 매수, RSI(4) > 50 → 매도"

### Donchian Channel (돈치안 채널)
- **뜻**: "최근 N일 중 최고가"와 "최근 N일 중 최저가"를 그린 박스.
- **사용**: "가격이 최근 20일 최고가를 뚫으면 강한 상승 신호 → 매수"
- **돌파(Breakout) 전략**: 박스 위로 뚫으면 산다, 아래로 뚫으면 판다.
- **우리 프로젝트**: Strategy A에서 "20일 최고가 돌파 시 매수, 10일 최저가 이탈 시 매도"

### ATR (Average True Range, 평균 진폭)
- **뜻**: 최근 N일 동안 하루 평균 가격 변동폭. "변동성" 측정.
- **예**: 비트코인 ATR(14) = 200만원 → "요즘 비트코인은 하루에 평균 200만원 정도 움직이네"
- **사용**: 손절 거리 계산에 씀. "현재가 - 2 × ATR"을 손절선으로 → 변동성 큰 시기엔 손절 멀리, 작은 시기엔 가까이.
- **왜**: 고정 % 손절(예: -3%)보다 시장 상황에 적응적.
- **자바 비유**: 동적 타임아웃 — 평균 응답 시간의 2배를 타임아웃으로 설정하는 것과 비슷.

### MACD (Moving Average Convergence Divergence)
- **뜻**: 두 개의 EMA 차이로 추세 변화 감지.
- **우리 프로젝트**: 안 씀 (EMA로 이미 같은 정보)

### 볼린저 밴드 (Bollinger Bands)
- **뜻**: MA를 가운데로, 위아래에 표준편차 ±2를 그린 밴드.
- **사용**: 가격이 상단 밴드 터치 = 과매수, 하단 = 과매도.
- **우리 프로젝트**: 일단 안 씀.

### 거래량 (Volume)
- **뜻**: 그 시간 동안 거래된 코인의 양.
- **사용**: "가격이 오르는데 거래량도 같이 오르면 진짜 상승. 거래량 없이 오르면 가짜."
- **우리 Strategy A**: "20일 평균 거래량의 1.5배 이상일 때만 매수"

### 캔들 / 봉 (Candle)
- **뜻**: 일정 시간 동안의 시가/고가/저가/종가/거래량을 묶은 데이터.
- **종류**:
  - **1분봉**: 1분 동안의 데이터
  - **4시간봉**: 4시간 동안의 데이터
  - **일봉**: 하루 동안의 데이터
- **자바 비유**: DTO 객체. `Candle { time, open, high, low, close, volume }`

### OHLCV
- **뜻**: 캔들 데이터의 5개 필드 약자
- **O**pen (시가), **H**igh (고가), **L**ow (저가), **C**lose (종가), **V**olume (거래량)

## 3. 백테스트 / 검증 용어

### 백테스트 (Backtest)
- **뜻**: "이 매매 규칙을 5년 전부터 썼다면 어떻게 됐을까?" 과거 데이터로 시뮬레이션.
- **자바 비유**: 단위 테스트 + 통합 테스트의 트레이딩 버전. 다만 **데이터에 결과가 의존**.

### 사전 지정 파라미터 (Pre-registered Parameters)
- **뜻**: "이 숫자들로 테스트할게" 미리 못 박고 시작.
- **반대 (나쁜 짓)**: 100가지 조합 다 돌려본 후 "최고 성과 나온 조합"을 발표 → 운빨인지 진짜인지 알 수 없음. 데이터 스누핑(data snooping).
- **자바 비유**: 단위 테스트에 expected 값을 미리 정해놓고 assertion. 결과 보고 expected 바꾸면 부정행위.

### Walk-Forward Analysis (워크 포워드 분석)
- **뜻**: 과거 데이터를 시간순으로 작은 조각들로 나눠서:
  1. 첫 1년 데이터로 매매 규칙 만들기
  2. 그다음 6개월 데이터로 그 규칙 테스트
  3. 윈도우를 6개월 미루고 반복
- **목적**: "과거에 통한 규칙이 미래(=다음 6개월)에도 통하는지" 확인.
- **자바 비유**: K-fold cross validation의 시계열 버전.

### 데이터 스누핑 (Data Snooping) / 곡선 맞춤 (Curve Fitting) / 과적합 (Overfitting)
- **뜻**: 과거 데이터에 지나치게 맞춰진 규칙. 미래엔 안 통함.
- **예**: "비트코인이 화요일 오전 11시에 만든 규칙이 최고!"라고 하지만, 그건 그냥 과거 데이터 우연의 일치.
- **자바 비유**: 머신러닝의 오버피팅. 학습 데이터엔 100% 맞지만 새 데이터엔 50% 정확도.

### Monte Carlo Simulation (몬테카를로 시뮬레이션)
- **뜻**: 거래 순서를 무작위로 섞어서 1000번 다시 시뮬레이션.
- **목적**: "운 좋게 좋은 순서로 거래된 거 아닌가?" 확인. 무작위 시뮬레이션 중 절반이 손실이면 운빨.

### Bootstrap (부트스트랩)
- **뜻**: 데이터를 무작위로 재추출해서 통계 신뢰구간 계산.
- **목적**: "샤프지수가 1.0인데 신뢰할 만한가?" 답하기 위해. 부트스트랩으로 "샤프 1.0 ± 0.3" 같은 범위 산출.

### Sharpe Ratio (샤프 지수)
- **뜻**: "위험 1단위당 얼마나 수익을 냈나" 점수. 높을수록 좋음.
- **계산**: (수익률 - 무위험 수익률) ÷ 수익률의 표준편차
- **해석**:
  - 0 미만: 손실 또는 돈 안 됨
  - 0~1: 그저 그럼
  - 1~2: 좋음
  - 2 이상: 매우 좋음 (의심스러울 정도)
  - 3 이상: 거의 사기, 과적합 의심
- **우리 목표**: 백테스트 1.0+, 라이브 0.5+

### Sortino Ratio (소르티노 지수)
- **뜻**: 샤프와 비슷하지만 "하방 변동성"만 분모로 씀. "오를 때의 변동성"은 페널티 안 줌.
- **왜**: 트레이더는 "오르는 거"는 변동성이라도 좋아함. 하방만 위험.

### Calmar Ratio (칼마 비율)
- **뜻**: 연수익률 ÷ 최대 낙폭(MDD). "고통 대비 수익".
- **예**: 연 50% 벌었지만 MDD 50% → 칼마 1.0. 연 20% 벌었지만 MDD 10% → 칼마 2.0 (후자가 더 좋음).

### MDD (Maximum Drawdown, 최대 낙폭)
- **뜻**: 백테스트 기간 중 최고점 대비 가장 큰 손실 폭.
- **예**: 1억 → 1억5천 (최고점) → 9천만원 → MDD = (15000-9000)/15000 = 40%
- **중요**: MDD 30%면 "이 기간 중 최악일 때 30% 잃었음" — 심리적으로 버틸 수 있나?
- **자바 비유**: SLA 위반 시 최대 다운타임. 평균이 좋아도 최악이 1주일이면 못 씀.

### Profit Factor (이익 팩터, PF)
- **뜻**: (총 이익 합) ÷ (총 손실 합)
- **해석**:
  - 1 미만: 손실
  - 1.0~1.3: 마진 얇음
  - 1.3~1.8: 양호
  - 1.8~3.0: 좋음
  - 3 이상: 의심스러움 (과적합?)

### Win Rate (승률)
- **뜻**: 전체 거래 중 이익 본 거래의 비율.
- **주의**: 승률 70%여도 손실 거래 하나가 너무 크면 망함. 승률만으로 판단 X.

### R-Multiple
- **뜻**: 한 거래의 수익 ÷ 그 거래의 리스크.
- **예**: 100만원 리스크하고 300만원 벌면 3R. 100만원 리스크하고 200만원 잃으면 -2R.
- **목표**: 평균 R-Multiple > 1 (즉, 평균적으로 리스크보다 더 벎)

### DSR (Deflated Sharpe Ratio, 디플레이티드 샤프)
- **뜻**: "100가지 조합 중에서 최고 샤프"를 그대로 발표하면 운빨이 섞여 있음. DSR은 "여러 시도를 보정해서 진짜 샤프"를 산출.
- **임계값**: DSR > 0 = 통계적으로 유의미. DSR > 0.5 = 우리 프로젝트 최소 기준.
- **자바 비유**: 다중 가설 검정에서의 본페로니 보정.

### CPCV (Combinatorial Purged Cross-Validation)
- **뜻**: ML 모델 검증 방법. 시계열 데이터에서 학습/테스트 분리 시 데이터 누수 방지.
- **우리 프로젝트**: 룰 기반이라 일단 안 씀. ML 단계에서만 필요.

### Live Degradation (실전 성능 저하)
- **뜻**: 백테스트 샤프가 1.5여도 실전에서는 0.6~0.9로 떨어짐 (30~60% 저하).
- **이유**: 백테스트는 슬리피지/지연/시장 충격 과소평가, 시장은 과거와 다르게 변함.

## 4. 위험 관리 / 성과 측정

### Position Sizing (포지션 사이징)
- **뜻**: "이번 거래에 얼마를 걸 것인가?" 결정.
- **고정 비율**: 항상 자본의 10%
- **변동성 조정 (ATR 기반)**: 변동성 큰 코인은 적게, 작은 코인은 많이

### Risk per Trade (거래당 리스크)
- **뜻**: 한 거래에서 최악의 경우 잃을 금액 = 자본의 X%
- **예**: 자본 100만원, 리스크 1% → 거래당 최대 1만원 손실 → 손절선까지 거리가 5%면 포지션은 20만원

### Drawdown (낙폭)
- **뜻**: 최고점 대비 현재 손실. MDD는 그 중 최대값.

### Circuit Breaker (서킷 브레이커)
- **뜻**: 시장 급락 시 봇을 자동으로 멈추는 안전장치.
- **자바 비유**: Hystrix/Resilience4j의 서킷 브레이커와 동일 개념. "에러 발생률 N% 이상이면 호출 차단".

### Kill Switch (킬 스위치)
- **뜻**: 사람이 수동으로 봇을 즉시 정지시키는 빨간 버튼.
- **자바 비유**: Spring Boot Actuator `/actuator/shutdown` 같은 거.

### Reconciliation (리컨실리에이션, 정합성 회복)
- **뜻**: 봇이 죽었다 살아났을 때 "지금 내 포지션이 정확히 뭐지?" 거래소에 다시 물어봐서 상태 동기화.
- **자바 비유**: 분산 시스템 재시작 후 state recovery.

- **예**: 업비트 BTC = 1억원, 바이낸스 BTC = $66,000 (≈ 9500만원). 차이 5% = 김치프리미엄 5%.
- **의미**: 김프 높음 = 한국 매수세 강함 = 종종 과열 신호.

### 도미넌스 (Dominance)
- **BTC 도미넌스**: 전체 크립토 시총에서 비트코인 비중
- **사용**: BTC 도미넌스 상승 = 자금이 BTC로 몰림 = 알트코인 약세

### 공포탐욕지수 (Fear & Greed Index)
- **뜻**: 시장 심리를 0~100으로 표현 (alternative.me 제공)
- **0~25 극도의 공포**: 폭락 후, 매수 기회일 수 있음
- **75~100 극도의 탐욕**: 거품, 매도 고려

### 알트코인 (Altcoin)
- **뜻**: 비트코인 외 모든 코인 (이더리움, XRP, 솔라나 등)

### 스테이블코인 (Stablecoin)
- **뜻**: 달러 등에 가치가 고정된 코인 (USDT, USDC, DAI)

### 스캠코인 (Scam Coin)
- **뜻**: 사기 목적의 코인. 거래량 조작, 유동성 부족, 갑자기 0원 됨.
- **우리 프로젝트**: 일일 거래량 10억원 미만, 시총 $50M 미만, 상장 6개월 미만 → 자동 제외.

### 러그풀 (Rug Pull)
- **뜻**: 개발자가 코인 만든 후 자금 들고 잠적. 가격 0원.

### 펌프앤덤프 (Pump & Dump)
- **뜻**: 작전 세력이 가격 인위적으로 띄운 후 팔아치워서 일반인 손해 보게 함.

### FOMC
- **뜻**: 미국 연방공개시장위원회. 미국 금리 결정 회의 (1년 8회).
- **영향**: 금리 인상 → 위험자산(크립토 포함) 하락 경향. 인하 → 상승 경향.

### CPI
- **뜻**: Consumer Price Index, 소비자물가지수. 인플레이션 측정.
- **영향**: CPI 높음 → Fed 금리 인상 압박 → 크립토 약세 경향.

### DXY (Dollar Index, 달러 인덱스)
- **뜻**: 달러가 다른 주요 통화 대비 얼마나 강한가.
- **영향**: DXY 강세 → 크립토 약세 (역상관 -0.7~-0.85)

### 온체인 데이터 (On-chain Data)
- **뜻**: 블록체인에 직접 기록된 거래 데이터. (지갑 주소, 거래량, 해시레이트 등)
- **예**: "고래 지갑이 거래소로 1000 BTC 보냄 → 매도 임박?"

### 해시레이트 (Hashrate)
- **뜻**: 비트코인 채굴자들의 총 연산 능력.
- **사용**: 해시레이트 급락 = 채굴 기업 도산 = 시장 약세 신호일 수 있음.

## 9. Week 1 추가 용어 (Day 0 감사 반영)

### Wilder Smoothing (와일더 스무딩)
- **뜻**: J. Welles Wilder가 고안한 지수 스무딩 방식. 표준 EMA보다 변화에 둔감.
- **사용**: RSI, ATR, ADX 같은 지표의 표준 계산법.
- **자바 비유**: 이동평균에 inertia를 더 많이 주는 EMA 변형.
- **우리 프로젝트**: `ta` 라이브러리의 ATR/RSI는 Wilder 스무딩 사용. 직접 SMA로 ATR 계산 금지.

### Padysak / Vojtko (저자명)
- **뜻**: Quantpedia 연구 그룹의 두 연구자. BTC 트렌드+평균회귀 앙상블 전략 논문 저자.
- **논문**: "Revisiting Trend-following and Mean-reversion Strategies in Bitcoin" (2022, 2024)
- **결과**: BTC 일봉 백테스트 Sharpe 1.71, CAGR 56% (논문 원본 값, 라이브 결과 아님)
- **우리 프로젝트**: 이 논문에서 영감 받음. **"복제"가 아닌 "영감을 받음"** (4시간봉, 업비트 KRW 등 변경점 있음)

### Walk-Forward Analysis (워크 포워드 분석)
- **뜻**: 시계열 데이터를 시간순으로 train/test 윈도우로 나누어 검증하는 방법.
- **예**: 1년 학습 → 6개월 테스트 → 윈도우 6개월 미루고 반복
- **자바 비유**: K-fold cross validation의 시계열 버전 (랜덤 셔플 금지).
- **우리 프로젝트**: Week 2에 도입. Week 1은 in-sample만.

### CPCV (Combinatorial Purged Cross-Validation)
- **뜻**: López de Prado가 제안한 시계열 ML 검증법. Walk-Forward의 발전형.
- **특징**: 여러 train/test 분할 + purging(인접 데이터 제거) + embargoing(테스트 직후 데이터 제외)
- **우리 프로젝트**: ML 단계(Phase 10+)에서만. Week 1~9는 단순 walk-forward.

### Politis-Romano Stationary Bootstrap
- **뜻**: 시계열 데이터의 자기상관을 보존하면서 부트스트랩 샘플링하는 방법.
- **사용**: Sharpe 신뢰구간 계산, "이 Sharpe가 우연인지 진짜인지" 검증.
- **우리 프로젝트**: Week 2 DSR 계산 시 사용.

### Almgren-Chriss Model
- **뜻**: 주문이 시장 가격에 미치는 영향(market impact)을 수학적으로 모델링한 공식.
- **사용**: 슬리피지 추정. `slippage_bps ≈ k * (order_size / bar_volume)^0.5`
- **우리 프로젝트**: Week 2+ 호가창 기반 슬리피지 모델에서 사용 가능.

### Donchian Channel (돈치안 채널) - 보강
- **뜻**: 최근 N일 최고가/최저가로 만든 박스. 1970년대 Richard Donchian이 고안.
- **돌파(Breakout) 전략**: 박스 상단 돌파 시 매수, 하단 이탈 시 매도.
- **우리 Strategy A**: 일봉 20일 최고가 돌파(매수), 10일 최저가 이탈(매도)
- **유래**: "Turtle Trading" 시스템의 핵심 지표. Richard Dennis가 사용해 유명해짐.

### Pre-registered Parameters (사전 지정 파라미터)
- **뜻**: 백테스트 시작 전에 "이 숫자들로 테스트할게" 미리 못 박은 파라미터 세트.
- **반대**: 100가지 조합 다 돌려본 후 최고값 보고 (data snooping)
- **자바 비유**: 단위 테스트의 expected 값을 미리 정해놓는 것.
- **우리 프로젝트**: Week 1 Go/No-Go는 사전 지정 파라미터 결과만 사용. 민감도 그리드는 참고용.

### Replication Sprint (복제 스프린트)
- **뜻**: 학술 논문 또는 공개 백테스트 결과를 우리 환경에서 재현 시도하는 단기 프로젝트.
- **목적**: "이게 진짜 작동하는가?" 빠르게 답하기.
- **우리 프로젝트**: Week 1~3 = Padysak/Vojtko 영감 전략의 업비트 KRW-BTC 복제 시도.

### Data Snooping (데이터 스누핑) - 보강
- **뜻**: 같은 데이터로 여러 가설/파라미터를 시도한 후 최고 결과만 발표하는 행위.
- **자바 비유**: 단위 테스트를 100번 무작위로 돌리고 우연히 통과한 케이스만 보여주기.
- **위험**: 100조합 중 1개는 운빨로도 좋게 나옴. 그 1개를 골라 발표하면 사실상 거짓.
- **방지**: 사전 지정 파라미터, Deflated Sharpe Ratio, 다중 검정 보정.

### vectorbt 0.28.5 — 주요 파라미터 (검증됨)
- `Portfolio.from_signals(close, entries, exits, sl_stop, sl_trail, tp_stop, freq, init_cash, fees, slippage, group_by, cash_sharing)`
- `sl_stop`: 손절 fraction (0.05 = 5%)
- `sl_trail=True`: sl_stop을 트레일링으로 동작 (별도 ts_stop 파라미터 없음)
- 시간 스톱: vectorbt 무료 버전에 없음. `entries.shift(N)` 사전 계산해서 exits에 OR
- `pf.sharpe_ratio()`: 메서드 (괄호 필수)
- 앙상블: `cash_sharing=True + group_by=True` + 2-column DataFrame

### pyupbit 0.2.34 — 주요 함수 (검증됨)
- `get_ohlcv_from(ticker, interval, fromDatetime, to, period)`
- `to=` 파라미터 존재함
- `interval`: `"day"`, `"minute240"`, `"week"` 등
- 인덱스: timezone-naive **KST** (UTC 아님) → `tz_localize('Asia/Seoul').tz_convert('UTC')` 필수
- 에러 시 `None` 반환 (예외 없음). 재시도 wrapper 필요.
- `period=0.2` 권장 (기본 0.1은 Upbit rate limit 한계)
