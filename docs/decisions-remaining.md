## Q4 (재질문). 대시보드 보안

### 당신이 말한 것
> "RSA 암호화로 비밀번호 암호화해서 서버에서 복호화해서 체크"

### 기술적 현실 — RSA + HTTP의 한계

말씀하신 방식은 실제로 일부 은행이 사용하는 기법이고 구현 가능합니다. 다만 **HTTP만 쓰면 남는 구멍 3가지**:

#### 구멍 #1: 세션 쿠키/JWT는 여전히 평문

```
로그인 과정:
  1. 클라이언트 → 서버: 공개키 요청
  2. 서버 → 클라이언트: 공개키 전송
  3. 클라이언트: 비밀번호를 공개키로 암호화 → 전송  ✅ 안전
  4. 서버: 개인키로 복호화 → 검증 → 세션 토큰 발급
  5. 서버 → 클라이언트: 세션 토큰 전송 (평문)  ❌ 노출
  6. 이후 모든 요청: 세션 토큰 첨부 (평문)  ❌ 노출
```

→ 공격자는 비밀번호 몰라도 **세션 토큰만 훔치면 로그인 우회**

#### 구멍 #2: 공개키 변조 (MITM)

```
공격자가 중간에 있으면:
  클라이언트 ↔ [공격자] ↔ 서버

  클라이언트가 공개키 요청 →
  공격자가 가로채서 자기 공개키로 바꿔치기 →
  클라이언트는 공격자의 공개키로 암호화 →
  공격자가 복호화해서 비밀번호 획득 →
  다시 서버의 진짜 공개키로 암호화해서 전달
```

→ HTTP는 "서버가 보낸 공개키가 진짜인지" 검증 불가. HTTPS는 인증서(CA)로 이걸 막음.

#### 구멍 #3: 대시보드 조작 명령도 평문

```
로그인 성공 후:
  "전 포지션 청산" 버튼 클릭 →
  POST /api/emergency-liquidate (평문)  ❌
  API 응답 (포지션, 수익률 등) (평문)  ❌
```

→ 공격자가 민감 정보 스니핑 + 명령 주입 가능

### 결론

**RSA 암호화는 "비밀번호 하나"만 지키고, 나머지 전부(세션, 데이터, 명령)는 여전히 노출.**

### 옵션 재정리

| 옵션 | 전송 암호화 | 설정 난이도 | 개발 시간 | 외부 접속 | 완전성 |
|------|:-----------:|:----------:|:--------:|:---------:|:------:|
| **A. Cloudflare Tunnel** ⭐ | HTTPS (자동) | 20분 | 0 | ✅ 공식 URL | ⭐⭐⭐⭐⭐ |
| B. Tailscale | VPN (자동) | 10분 | 0 | ✅ VPN 접속 | ⭐⭐⭐⭐⭐ |
| C. RSA + HTTP + 짧은 JWT + IP 바인딩 | 부분 | 중간 | 수 시간 | ✅ 직접 IP | ⭐⭐ |

### 각 옵션 상세

#### Option A. Cloudflare Tunnel (가장 추천)

**동작 방식**:
- 로컬 PC에 `cloudflared` 데몬 설치 (Docker 컨테이너 가능)
- 봇 대시보드(예: localhost:3000)가 Cloudflare의 글로벌 엣지에 자동 연결
- Cloudflare가 `https://xxxx-xxxx.trycloudflare.com` 같은 무료 HTTPS URL 제공
- 또는 도메인 있으면 `https://coinbot.yourdomain.com` 사용 가능

**장점**:
- 도메인 불필요 (자동 trycloudflare.com)
- TLS 인증서 자동 관리
- 로컬 PC에 **포트를 전혀 열 필요 없음** (아웃바운드만) → 방화벽 설정 0
- 무료 플랜에서 Cloudflare Access 추가 가능 (이메일 OTP, Google SSO, GitHub 로그인 등)
- 월 비용 $0
- 모바일 브라우저에서 바로 접속

**단점**:
- Cloudflare 계정 필요 (무료 가입)
- trycloudflare.com URL은 재시작 시 바뀔 수 있음 (고정 원하면 무료 도메인 연결)

**구체 설정 단계**:
1. cloudflared 설치 (brew install cloudflared)
2. `cloudflared tunnel login` (브라우저에서 Cloudflare 로그인)
3. `cloudflared tunnel create coinbot`
4. 설정 파일 1개 작성 → 로컬 dashboard 포트 매핑
5. `cloudflared tunnel run coinbot` → URL 발급
6. 선택: Cloudflare Zero Trust Access로 로그인 방식 설정

#### Option B. Tailscale

**동작 방식**:
- 사용자의 모든 기기(PC, 폰, 노트북)에 Tailscale 앱 설치
- 기기들이 "개인 VPN 네트워크"로 연결됨
- 대시보드는 `http://coinbot-pc.tailnet-name.ts.net:3000` 같은 내부 주소로 접속
- 외부 인터넷에는 노출 0

**장점**:
- 개인용 무료 (최대 100대 기기)
- 설정 진짜 10분
- 모바일 앱 있음 → 외출 시 Tailscale 켜면 집 PC 접속 가능
- 개인 기기만 접속 가능 (공격 벡터 극소)

**단점**:
- 접속하려는 모든 기기에 Tailscale 설치 필요
- 다른 사람(가족)이 자기 폰에서 접속하려면 그 폰에도 Tailscale 설치

#### Option C. RSA + HTTP + 짧은 JWT + IP 바인딩 (원하신 방식)

**동작 방식**:
- 클라이언트: JavaScript로 RSA 공개키 가져와서 비밀번호 암호화 후 전송
- 서버: 개인키로 복호화, bcrypt로 비교
- 성공 시 JWT 발급 (TTL 15분, 주기적 갱신)
- JWT에 발급 시점 IP 포함 → 요청 IP 일치해야 통과

**장점**:
- 외부 도구 의존 없음
- 요청하신 방식과 일치

**단점**:
- 세션 쿠키/JWT 평문 노출 문제 완전 해결 안 됨 (IP 바인딩으로 완화)
- 공개키 변조(MITM) 취약 (도구 없이 완벽 방어 불가)
- 개발 시간 수 시간 (RSA 라이브러리, 키 관리, JWT, IP 체크 전부)
- 민감 데이터(포지션, 수익률) 전송 시 평문

## Q9 (재질문). LLM Quota / 계정 전략

### 당신이 말한 것
> "API 키 발급하고 싶지 않아. Anthropic 플랜 2개 + ChatGPT 있으니 계정 전환 가능하게 해줘."

### 기술적 현실

1. **Claude Code CLI (OAuth Max 플랜)**은 **한 번에 하나의 계정만** 로그인 가능
2. **봇 내부에서 OAuth 토큰을 직접 관리하는 것은 비공식** — Anthropic이 공식 API가 아닌 CLI용 토큰을 직접 호출하는 방식은 언제든 막힐 수 있음
3. **ChatGPT Plus는 API 키 별개** — Plus 구독은 웹 UI용, 봇에서 쓰려면 OpenAI API 키 결제 필요 (월 $20 구독과 별개의 토큰 과금)

→ "결제 없이 2계정 자동 전환"은 **현재 기술적으로 불안정**

### 대안 옵션

#### Option A. Quota 모니터링 + 초과 시 LLM 기능 자동 중단

**동작 방식**:
- 봇이 LLM 호출 전 rate limit 헤더 체크
- 80% 초과 시 warning, 95% 초과 시 해당 기능 일시 중단
- 룰 기반 전략은 **계속 정상 동작**
- 디스코드 알림 "LLM 레이어 일시 중단 중"
- 5시간 창 리셋되면 자동 재개

**장점**:
- 구현 단순
- 룰 기반은 영향 없음 (돈 버는 메인 로직은 계속)
- 유료 결제 불필요

**단점**:
- LLM이 잠시 꺼질 수 있음

#### Option B. 매우 보수적 호출 스케줄

**동작 방식**:
- 주간 1회 Opus (일요일 코인 화이트리스트 + 전략 리뷰)
- 일일 1회 Sonnet (매크로 이벤트 태깅 + 전일 성과 요약)
- **총 일일 ~2콜**, 월간 ~62콜
- Max 20x 5시간 창 한도의 <1% 사용
- 사용자의 개발/일상 Claude 사용 방해 0에 가까움

**장점**:
- 사실상 quota 문제 없음
- 개발 작업과 충돌 없음

**단점**:
- 매 4시간 호출은 포기 (하지만 4h 봉 기준 실익 크지 않음)

#### Option C. Phase 10+에서 결정

**동작 방식**:
- LLM 레이어는 베이스 전략 페이퍼 4주 통과 후 (Phase 10+)에만 추가
- 지금은 이 문제 결정 미룸
- Phase 10 도달 시점에 상황(계정 상태, API 가격, 필요도) 재평가

**장점**:
- 지금 결정할 필요 없음
- Phase 10까지 수개월 걸리므로 그동안 상황 변화 가능
- **Q5에서 이미 "A. Phase 10+로 연기" 선택하셨으므로 자연스러운 흐름**

**단점**:
- 없음 (결정 미루기만)

#### Option D. 나중에 ChatGPT API 구독 고려

**동작 방식**:
- 지금은 Anthropic Max 20x만 사용
- Phase 10+에서 LLM 기능 추가 시, 사용량이 많으면 OpenAI API 구독 추가 ($5~20/월)
- 폴백용으로 활용

**비용**:
- OpenAI API pay-as-you-go, 월 $5~20 예상
- 지금은 결제 안 해도 됨
