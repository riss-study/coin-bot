## 🟠 보안 결정 (치명적)

### Q4. 대시보드 보안 재결정

원래 답변(C-3)에서 "HTTP + 외부 IP + 로그인"이라고 하셨는데, 이는 **로그인 비밀번호가 평문으로 인터넷 통과** = 기술적으로 안전하지 않습니다. API 키는 기업급으로 보호하면서 이 모순은 치명적.

- [ ] **A. Tailscale + localhost 전용 (강력 권장)**
  - 무료 개인 플랜, 설정 10분
  - 외부 포트 0개, 평문 0개
  - 폰/태블릿에서 Tailscale 앱 설치하면 어디서든 접속 가능 (사실상 "외부 접속"과 동일)
- [ ] **B. Cloudflare Tunnel + HTTPS + 2FA (TOTP)**
  - 무료 티어, 공식 HTTPS, 도메인 필요 없음
  - 2FA 앱 (Google Authenticator 등) 설치 필요
- [ ] **C. 로컬 전용 + Discord 원격 모니터링**
  - 대시보드는 집 PC에서만
  - 외출 시 조회/제어는 Discord 봇 명령어
- [ ] D. 직접 판단해줘

**답변:**
Discord 원격 모니터링은 당연히 가능하게 해줘. 그리고 내가 말한 로그인은 이 대시보드를 들어오기 위한 계정에 대한 로그인이야. 당연히 로그인할 때 로그인 비밀번호는 암호화해서 전달하게 해줘. RSA 암호화로 비밀번호 암호화해서. 그걸 다시 서버에서 복호화해서 체크하는 걸로. 무슨말인지 모르겠으면 말해줘.

### Q6. LLM 역할 재선택 (재검증 결과 반영)

**새 증거 요약**:
- ❌ Reddit 감성 (B-1a): **내 원래 근거 자체가 논문 오독**. 증거 없음 → 제외 권장
- ✅ **스케줄 매크로 이벤트 (재채택 권장)**: IMF/BIS/Fernandez-Fuertes 강한 학술 증거
- ✅ **LLM Validator (거부권만)**: Agent Market Arena의 "아키텍처 > 모델" 메시지에 부합
- ✅ 레짐 내러티브 (B-1c): 의사결정 영향 없음, 안전
- ✅ 사후 분석 (B-1f): 안전, 유용
- ✅ 주간 화이트리스트 (B-1g): 유용 (단, "LLM은 브레이크만" 원칙과 모순이므로 주의)
- ⚠️ 다중 에이전트: **로깅 하네스로만** (의사결정 아님)

권장 LLM 역할 세트:

  - **LLM Validator** (룰 신호 후 거부권만, proceed/skip/skip-hard)
  - 레짐 내러티브 (대시보드 해설용)
  - 사후 분석 리포트 (일/주간)
  - 주간 화이트리스트 리뷰 (단, 제외만 가능, 추가는 룰 기반)
