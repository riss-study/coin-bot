## Part 1: 사실 오류 — 논문을 잘못 읽었습니다

### 오류 #1: arXiv:2410.12464에 대한 치명적 오독

**제가 주장한 것 (틀렸음):**
> "Reddit 감성은 +23.3% 수익, 뉴스 감성은 성능 저하."

**논문이 실제로 말하는 것:**
- 이 논문은 **Reddit vs 뉴스를 비교하지 않음**. "Reddit-sourced"라는 프레이밍 자체가 논문에 존재하지 않음.
- 실제 비교 대상은 **"팩트 기반 뉴스 추론 vs 주관 기반 뉴스 추론"** — 둘 다 **뉴스 기반**.
- 논문의 실제 결과: FS-ReasoningAgent(뉴스 기반) → **BTC +7%, ETH +2%, SOL +10%** 수익 개선
- Ablation: 주관적 추론 제거 시 불장 수익 76.19% → 66.04% 감소. 팩트 추론 제거 시 베어마켓 -15.91% → -19.21% 악화.
- 논문 결론: **"뉴스는 더 나은 처리가 필요하지, 버려야 할 것이 아님."**

**영향**:
- B-1a(Reddit 감성)의 근거가 **증발함**. 논문 근거 없이 제가 만든 이야기였음.
- B-1b(뉴스 감성)를 "성능 저하" 이유로 거부한 것은 **오판**. 논문은 오히려 뉴스 감성이 개선됨을 보임.

**교정 방향**: Reddit 감성 분석은 **"증거 없는 실험"**으로 재분류. 뉴스/매크로 감성은 **오히려 증거가 더 강함**.

### 오류 #2: Agent Market Arena 체리피킹

**제가 주장한 것 (부분 진실이지만 체리피킹):**
> "GPT-5, Claude Sonnet 4 포함 대부분의 LLM이 BTC/ETH 바이앤홀드를 못 이김"

**논문이 실제로 보여주는 것:**
- 일부 에이전트 구성은 **크게 아웃퍼폼**:
  - InvestorAgent + GPT-4.1: **TSLA에서 40.83% 수익, Sharpe 6.47**
  - HedgeFundAgent on ETH: 일부 설정에서 **+39.66%** (하지만 다른 설정에선 큰 손실 — 고변동)
  - DeepFundAgent: **Sharpe 1.4~2.0** 일관된 균형 수익
- 논문의 **실제 핵심 주장**: *"에이전트 아키텍처가 지배적 요인이며, 기반 LLM 선택보다 중요하다."*
- 즉, "LLM이 실패"가 아니라 "스캐폴딩이 중요"가 실제 메시지.

**영향**: B-1e(직접 매매 결정)를 완전 거부한 것은 **과도한 결론**. "좋은 아키텍처 + LLM이 거부권(veto) 역할"이라는 중간 지점이 증거상 지지됨.

### 오류 #3: 스케줄된 거시경제 이벤트 분석의 증거 무시

**제가 놓친 증거:**
- **IMF Working Paper WP/2025/109**: 169개 중앙은행 74,882건 문서를 LLM으로 분석 → **향후 금리 움직임을 유의미하게 설명**. 대규모, 엄밀한 연구.
- **Fernandez-Fuertes (JMP 2025)**: Fed Beige Books + 회의록을 multi-agent LLM이 처리 → **수익성 있는 금리 커브 거래 전략** 생성.
- **BIS Working Paper 1215**: 중앙은행용 언어 모델(CB-LMs)의 정책 신호 추출 증명.
- **arXiv:2508.08001**: Fedspeak 해석의 불확실성 인식 프레임워크.

**핵심 구분**:
- **스케줄된 고밀도 텍스트 (FOMC, CPI, 규제 발표)** = LLM이 가장 잘하는 영역
- **연속 뉴스 피드 잡음** = LLM이 실패하는 영역
- 제가 이 둘을 묶어서 판단한 것이 실수

**영향**: B-1b(뉴스/거시 내러티브)를 거부한 것은 **증거와 반대**. FOMC 성명문, CPI 발표문 등을 Opus가 사전에 읽고 리스크 태그 생성하는 건 학술적으로 강하게 지지됨.

## Part 2: 전략적 오류 — "전략 우선"이라고 해놓고 인프라 우선으로 빌드

### 감사관 관찰

> **"사용자가 명시적으로 '전략 우선, 난 실질적으로 돈을 벌어야 한다'고 했는데, 결정 #1은 `nautilus_trader` 2~3주 학습, 결정 #6은 감사 로그 대시보드, 결정 #7은 다층 보안 키 관리. 업비트 데이터로 백테스트 한 번 돌리기 전에 ~1개월이 인프라에 소모됨."**

### 핵심 모순

사용자의 답변에서:
1. **"전략 우선"** (속도, E-4)
2. **"엄격한 테스트"** (신중, E-3)
3. **"단계별 승인"** (느림, F-1)
4. **"Sharpe>1.0 + DSR>0.5 + 페이퍼 4주"** (매우 높은 기준, D-2)

→ 이 4개는 **함께 성립 불가능**. 제가 이 충돌을 지적하지 않고 조용히 "인프라 우선 + 신중"으로 해석함.

### 현실 체크: $75~$750 규모에서 수학이 성립하는가?

**냉정한 계산:**
- 업비트 왕복 수수료 ≈ 0.1%
- 스윙 거래 목표 수익: 2~3%, 손절 1.5%
- 이익 Sharpe가 **backtest의 30~60%로 저하** (역사적 경험칙)
- Padysak/Vojtko의 Sharpe 1.71 → 실전 예상 **0.51~1.03**
- 사용자의 승진 기준 **Sharpe > 1.0** → **통과 확률 낮음**
- $75 원금에서 30% 수익 = $22. 이 돈을 벌기 위해 **매일 2시간 × 수개월** = 시급 몇 센트
- $750 원금이어도 의미 있는 절대 수익 어려움

**감사관 지적:**
> *"사용자는 '성장형'과 '실질적으로 돈을 벌어야'를 선택했다 — 그게 더 동기부여 되기 때문. 하지만 $75~$750에서 수학은 그걸 지지하지 않고, my-decisions.md의 어느 결정도 이 점을 지적하지 않는다."*

**가장 가능성 높은 실패 모드 (감사관 추정):**
- (a) **50% 확률**: 4~8주 인프라/프레임워크 작업 → 라이브 도달 전 동력 상실 → 프로젝트 중단
- (b) **25% 확률**: 라이브 도달 → 전략 저조 → 재튜닝 반복 → 과적합 → 손실
- (c) **5~10% 확률**: HTTP 대시보드 보안 사고
- (d) **5~10% 확률**: 전략 실제 작동, 자금 증액
- (e) **5% 확률**: 업비트 점검 시간 중 치명적 버그

## Part 3: 보안 모순 — 치명적

### 발견

**결정 #7 (API 키 보관)**: macOS Keychain + Docker + IP 화이트리스트 + 90일 로테이션 + 출금 권한 OFF → 기업급 엄격성

**vs.**

**결정 #6 (대시보드) + 사용자 답변 C-3**: 외부 IP로 HTTP 접근 허용 + 로그인 비밀번호

### 문제

- **대시보드 로그인 비밀번호가 평문으로 인터넷을 통과**
- **세션 쿠키도 평문으로 전송**
- 공격자가 같은 Wi-Fi, 같은 네트워크 세그먼트, 또는 중간 홉에서 **스니핑 가능**
- 봇을 조종할 수 있는 대시보드의 보안이 약하면 API 키 보호가 무의미
- 출금 권한이 없더라도 공격자가 **펌프앤덤프** 방식으로 사용자 자금 손실 가능
- 2단계 확인, 5초 취소 타이머 등은 **공격자가 비밀번호 알면 의미 없음**

### 필수 교정

- [ ] **Option A (권장)**: Tailscale / WireGuard VPN → 로컬호스트 전용 대시보드. 외부 포트 0, 평문 0
- [ ] **Option B**: Cloudflare Tunnel + HTTPS + 비밀번호 + 2FA (TOTP)
- [ ] **Option C**: 외부 접속 완전 포기 → 디스코드가 원격 모니터링, 위험한 액션은 로컬호스트만

**제가 어느 것도 제안하지 않은 것이 명백한 실수.**

## Part 4: 누락된 필수 결정들

### 누락 #1: 한국 과세 로깅 구현

사용자는 D-4에서 **Option 1 (DB 영구 저장 + 월별 CSV)**를 명시적으로 선택했지만, `my-decisions.md`에 **이 결정이 없음**. 스키마, 회계 방법(FIFO/평균), KRW 환산 기준, 수수료 처리 등 아무것도 정의 안 됨. 2025년 이후 과세 도입 시 **법적 필수**.

### 누락 #2: 데이터 파이프라인 / 업비트 히스토리 수집

- 업비트 REST API는 호출당 반환 개수 제한 → 5년치 데이터 수집에 수 시간~수일
- 데이터 갭 처리, 이상치 감지, 리샘플링 무결성, 백필 전략 → 아무것도 정의 안 됨
- "데이터 품질 99.9%"라고 선언만 하고 메커니즘 없음

### 누락 #3: 프로젝트 킬 크라이테리아

- "N주 후 DSR > 0.5 통과 못 하면 **중단하고 완전히 다른 접근** 또는 **학습 프로젝트로 재정의**"가 없음
- D-3은 "재검토"라고만 되어 있음 → 무한 재튜닝의 함정
- **퀀트 프로젝트가 실패하는 전형적 방식**: 출구 없이 무한 반복

### 누락 #4: 업비트 특화 운영 리스크

- 상장폐지 공지 처리
- 점검 시간 중 봇 동작 (업비트 정기 점검은 긴 편)
- KRW 입출금 동결
- API 다운타임 중 포지션 유지
- 알트코인 부분 체결

### 누락 #5: 키 로테이션 실행 절차

"90일 로테이션"이라고 적혀 있지만:
- 포지션 보유 중 로테이션 처리 방법?
- 봇 중단 → 키 교체 → 재시작 → 미체결 주문 재조정 절차?
- 이것이 없는 "로테이션" = **가동 중단 사고 예약**

### 누락 #6: 기타

- 대시보드 로그인 인증 메커니즘 상세
- NTP 동기화 (업비트 API 서명 요청은 시간 오차에 민감)
- 재해 복구 (PC 분실/고장 시 오픈 포지션 처리)
- 트레이드별 최대 손실 vs 포트폴리오 최대 손실 구분
- 업비트 API rate limit 관리

## Part 5: 모순 발견

### 모순 #1: "LLM은 브레이크 페달만"이라면서 B-1g는 가속 페달

결정 #4 원칙: *"LLM은 브레이크만, 가속 페달 아님"*
동시에 결정 #2의 B-1g: *"LLM이 주간 코인 화이트리스트 리뷰"*

→ 화이트리스트에 코인을 추가하는 것은 **실질적으로 매수 기회 생성** = 가속 페달. 자기기만.

### 모순 #2: 앙상블이 통과하지만 개별이 탈락하면?

결정 #8: "두 전략 백테스트 후 DSR > 0.5 둘 다 통과해야 별 차이 없음 판정"
D-3: "모든 전략이 DSR > 0.5 미달 → 재검토"

→ **앙상블은 통과하는데 개별 전략이 탈락하는 케이스**에 대한 정의 없음. 앙상블의 특성상 개별보다 앙상블이 더 잘 나오는 게 정상인데, 이 케이스의 로직이 비어 있음.

### 모순 #3: 인터넷 단절 "복구 대기" vs 손절 실행

C-1 답변: *"인터넷 끊김 → 복구까지 대기"*
앙상블 전략: *"ATR 기반 손절, 서킷 브레이커"*

→ 크래시 중 인터넷 끊기면 손절 실행 불가 = **최대 낙폭 목표 달성 불가**. 이 리스크 인정 없음.

### 모순 #4: Claude Max 20x 격리

결정 #3: "일일 ~8 LLM 호출, Max 20x 여유 충분"
**빠진 고려**: 사용자의 Claude Max는 **이 개발 작업 자체**에도 쓰임. 봇이 버그로 루프 돌면 사용자의 다른 작업 중 quota 소진 가능. 봇 전용 별도 API 키 (Anthropic Console) 사용 여부 결정 없음.

## Part 6: 솔직한 재검토 — LLM B-1 항목 재평가

사용자의 질문에 직접 답변: **"직접 매매, 뉴스 감성, 다중 에이전트 제외가 맞는가?"**

### B-1e (직접 매매 결정) — 여전히 반대, 하지만 이유 수정

**제가 틀렸던 부분**: "모든 LLM이 실패"라는 주장은 체리피킹.
**여전히 맞는 부분**: 크립토 4시간봉 스윙 전용 라이브 벤치마크는 **0건**. 스케일링이 확실치 않음.

**조정된 권고**:
- ❌ LLM이 직접 "매수/매도" 결정 생성
- ✅ **"LLM as Validator"** — 룰 기반이 후보 진입 신호 생성 → Opus가 {proceed, skip-with-reason, skip-hard} 답변
- 거부권만, 절대 새 진입 생성 안 함
- 50개 트레이드 후 오프라인 거부 정밀도 측정 → 랜덤 이하면 비활성화
- 이것은 제가 이전에 거부했던 것보다 **증거에 더 근접한 접근**

### B-1b (뉴스/거시 감성) — 입장 반전

**제가 틀렸던 부분**: arXiv:2410.12464를 오독. 그 논문은 뉴스 감성이 실제로 작동함을 보임.

**조정된 권고**:
- ❌ 연속 크립토 뉴스 피드 스크레이핑 → 잡음 많음 (부분적으로 맞음)
- ✅ **스케줄된 거시 이벤트 모듈**: FOMC 성명, CPI 발표, 주요 규제 조치 → Opus가 사전에 읽고 구조화 태그 생성 `{risk-off, neutral, risk-on} + confidence + horizon`
- 증거: IMF WP/2025/109, Fernandez-Fuertes 2025, BIS WP1215 모두 강한 학술 지지
- 이 태그는 **룰 기반 레짐 게이트의 입력**이지 직접 신호 아님

### B-1d (다중 에이전트) — 부분 수정

**제가 틀렸던 부분**: 가치 전무라고 판단.
**여전히 맞는 부분**: 의사결정 품질 향상 증거는 약함.

**조정된 권고**:
- ❌ TradingAgents를 의사결정 엔진으로 채택
- ✅ **다중 역할 로깅 하네스**: 룰 기반이 실행, LLM 역할들이 사후 설명
  - "Analyst" Agent: 지표 해석을 구조화된 이유로 기록
  - "Risk" Agent: 룰 기반 한도에 대한 체크 (거부권)
  - "Reviewer" Agent: 종가 마감 트레이드 주간 회고
- 가치: **감사 추적, 설명 가능성, 디버그 용이성** (PnL 개선 아님)
- 비용: Opus 주간/월간, Sonnet 트레이드당 로그

**사용자가 "다른 사람이 만든 걸 검증한다는 생각으로"라고 한 것이 결정적이었습니다.**

- ✗ 한 논문(arXiv:2410.12464)을 **완전히 오독**
- ✗ 다른 논문(Agent Market Arena)을 **체리피킹**
- ✗ 매크로 LLM 분석에 대한 **강한 학술 증거를 놓침** (IMF, BIS)
- ✗ "전략 우선"이라고 선언해놓고 **인프라 우선 빌드**
- ✗ 기업급 API 키 보안 vs **HTTP 평문 대시보드 모순** 무시
- ✗ **$75~$750에서 수학이 안 맞는다**는 사실을 사용자에게 솔직히 말하지 않음
- ✗ 프로젝트 **출구 기준 없음** → 무한 재튜닝 함정

이 교정안이 좀 괴로울 수 있지만, **실제 돈이 걸렸을 때 솔직한 게 사용자를 가장 돕는 길**이라고 생각합니다.
