# 외부 감사관 관점 자가 재검증

## 페르소나 전환

방금 응답을 **다른 사람(예: ChatGPT 또는 외부 컨설턴트)이 작성했다고 가정**하고 비판적으로 재검증. rubber-stamp 절대 금지. 사용자 메모리 `feedback_verification.md` + handover #10/11 정신 적용.

## 라운드별 검증 (최소 5라운드, 가능하면 10+)

### 라운드 1 — 사실 정확성

- 인용한 라인 번호, 파일 경로, 함수 시그니처 직접 Read로 확인
- 수치 (SHA256, 비율, 카운트)가 출력 결과와 일치하는지 cross-check
- 외부 라이브러리 API 호출 → 공식 docs 또는 venv 소스 직접 확인 (CLAUDE.md "추측 금지" 원칙)
- 추측으로 단정한 부분 (예: "X는 RWA", "Y는 stable")이 진짜 검증됐는지

### 라운드 2 — 박제 정합성

- 인용한 박제 출처 (decisions-final.md, criteria, sub-plan 등) 실제 본문과 일치
- 박제 시점 (사용자 승인, freeze 발효)이 정확
- 다른 박제와 충돌 없는지 (cross-reference)
- 라벨/명칭/버전 라벨 동기화 (예: "v3 → v4" 변경 후 상태 라벨 미동기화 사례 다수)

### 라운드 3 — 누락 발견

- 박제 본문에 빠진 항목 (예: 시점 명시, 출처 인용, cross-reference)
- 사용자 결정 사항 누락 (예: Fallback 옵션 사용자 결정 받기 전에 진행)
- evidence 파일 미저장 / 미갱신
- handover 갱신 누락 (큰 박제 작업 후)

### 라운드 4 — 자기 모순

- 같은 사실이 두 곳에 다르게 박제
- 시제 (과거/현재/미래) 혼용
- 같은 결정의 라벨/명칭이 일관되지 않음 (예: cycle 1 vs v4, cycle 2 vs v5)
- 6.1/6.2 freeze 발효 시점 순환 정의

### 라운드 5 — cycle 1 학습 패턴 재발 검증

`.claude/handover-2026-04-17.md` "과거 반복된 버그 유형" #1~#20 패턴이 재발하지 않았는지:
- 측정 창 inclusive off-by-one
- Fallback 라벨 misnomer
- 박제 freeze 시점 순환
- 실측 cherry-pick 재유입
- sub-plan 박제 vs .gitignore 룰 충돌
- 외부 코인 정체 추측
- 수치 단위 표기 오류 (% vs 비율)
- 사전 지정 추정 리스트 빗나감

### 라운드 6+ (필요 시 추가)

- 외부 라이브러리 호출 정확성 (vectorbt, pyupbit, ta 등 — research/CLAUDE.md 검증된 패턴)
- 보안/타임존/권한 (CLAUDE.md Immutable 룰)
- 다음 사이클 위험 예측 (cherry-pick 통로, 자가 분할 통로 등)
- 사용자 위임 발화 박제 정직성

## 결과 형식

```
## 자가 OK 판정 — [YES / NO]

## 라운드 검증 결과
| 라운드 | 발견 | 처리 |
|--------|------|------|
| 1 사실 정확성 | ... | ✓ / 정정 필요 |
| 2 박제 정합성 | ... | ... |
| ... | ... | ... |

## 발견 사항

### CRITICAL (n건) — BLOCKING
- ...

### WARNING (n건) — 정정 권고
- ...

### NIT (n건) — 가독성/스타일
- ...

## 결정 (위임받은 경우)
- 발견 사항 정정 진행 / 사용자 결정 사항 보고 / 자가 OK 후 진행
```

## 사용자 핵심 정신

- **"팀장/이사급 책임"** (handover #10): APPROVED 판정 시 책임지고 내림. 단 실측으로 깨질 수 있는 가정이 남아있으면 투명 보고
- **"이상 없는 거야?" 재확인** (handover #11): APPROVED 후에도 사용자가 재확인 요청 시 "문서 품질 OK + 실측으로만 확정되는 가정" 정직하게 분리해서 답변
- **"NIT까지 전부 해소 후 진행"** (handover #208): cycle 1 학습. NIT 잔존도 정정 후 진행
- **"솔직하게"** (사용자 반복 요청): rubber-stamp 금지. 자가 발견을 정직하게 보고
