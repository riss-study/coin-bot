## 3. 비교 도구 (`compare_backtest_paper.py`) 설계

### 3.1 입력
- `--paper-trades` (default `engine/logs/trades-{YYYY}.jsonl`)
- `--backtest-portfolio` (선택, vectorbt `Portfolio.pkl`. 미지정 시 페이퍼 통계만)
- `--days` (최근 N일 페이퍼 trades 필터)
- `--tolerance` (default 0.30 = ±30%, v2 박제)

### 3.1.1 호출 환경 (CRITICAL 정정 2026-04-26)
- vectorbt는 engine/.venv **미설치** (production 의존성 분리 유지).
- 비교 도구는 vectorbt 의존 → `research/.venv` 환경에서 PYTHONPATH 설정 후 호출.
- 호출 예시: `cd ~/project/coin-bot && source research/.venv/bin/activate && PYTHONPATH=engine python -m engine.scripts.compare_backtest_paper ...`

### 3.2 처리
1. `load_paper_trades` — JSONL 파싱 + ts_utc 필터
2. `compute_paper_stats` — sell 거래의 realized_pnl_krw 누적 + win/loss
3. `load_backtest_portfolio_stats` — vectorbt Portfolio 로드 + `.stats()` 핵심 필드 추출
4. `compare` — 누적 return % ±tolerance 일치 검증
5. `format_report` — 텍스트 리포트

### 3.3 sanity 결과 (현재 trades-2026.jsonl, 백테스트 미지정)
```
[페이퍼]
  trades:        1
  realized PnL:  +269 KRW
  invested:      99,948 KRW
  return:        +0.27%
  win/loss:      1 / 0

[백테스트] 미제공 — --backtest-portfolio 옵션으로 .pkl 경로 지정 필요.
verdict: no_backtest_portfolio_provided
```
(기존 V2-04 sanity 시점 1건 trade — 평가용 더미)
